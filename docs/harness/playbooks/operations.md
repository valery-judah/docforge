# Operations Playbook

## When To Use
Use for tooling, environment, release, maintenance, or operational work.

## Expected Docs To Touch
- Workstream index
- Evidence with commands run and outcomes
- Evergreen runbook or ADR if the operational change becomes durable

## Minimum Evidence Expected
- Operational change made
- Checks run
- Impacted environments or workflows

## Typical Exit Criteria
- Operational task is validated
- Recovery or troubleshooting notes are captured if needed
- Durable runbook guidance is updated when the process stabilizes
