# Refactor Playbook

## When To Use
Use for structural or maintainability changes with no intended product behavior change.

## Expected Docs To Touch
- Workstream index
- Evidence showing behavior preservation checks
- ADR only if the refactor changes durable architecture boundaries

## Minimum Evidence Expected
- What was reorganized
- How behavior preservation was checked
- Any follow-up cleanup still required

## Typical Exit Criteria
- Target structure is in place
- Existing behavior remains validated
- Remaining debt is captured explicitly
