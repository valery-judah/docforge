# ADR Template

## Purpose
Use this file when a decision becomes durable, cross-cutting, or important enough to stand outside a single workstream.

## When To Use
Create an ADR after the decision is clear and the context has been captured in the relevant workstream.

## Starter Structure
```md
# ADR-0001: [Title]

- Status: proposed
- Date: YYYY-MM-DD

## Context
What problem or pressure led to this decision?

## Decision
What was chosen?

## Consequences
- Positive consequence
- Negative or tradeoff consequence

## Related Workstreams
- `docs/workstreams/WS-000-example/`
```
